The model underneath mockingpug. Read these if you want to understand
*why* it behaves the way it does, not just how to call it.

- [Schema DSL](/docs/concepts/schema-dsl): every generator type an entity's data block can use.
- [Relations & Generation Order](/docs/concepts/relations-and-generation): field-level vs. bare cross-entity relations, and how generation order is decided.
- [Reconciliation & Storage](/docs/concepts/reconciliation-and-storage): how mockingpug decides what to (re)generate, and where data lives.
- [Pagination](/docs/concepts/pagination): page / offset / cursor strategies, query param names, and response shape.
